Manual Commit.

It is more flexible to make a commit manually. One can decide when to make a logical or reasonable commit.

Auto saving just save time, but it is hard to control when the commit would have been made.

For a functional project, like a web page, all the files are working together.

So it's very important to make commits comprise all the files involved in the project.

This method could make very single commit a fully working version of this project.

Git log command can reveal all the commits at a time. When the commit made, what was changed in it, who made this, more importantly, we can have the commit ID in the log.

Git diff can compare two commits line by line. It's easy to find any revisions with git diff.
